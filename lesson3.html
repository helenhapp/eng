What are Conditionals?

Think of conditionals like an if/else statement in programming! They have two parts: the If-clause (the condition or hypothesis), which begins with the word if, and the main clause, which shows the result.

When the if-clause comes first, we separate the two parts with a comma. If the main clause comes first, we don't need a comma.

  • If the code is correct, the program will run.
  • The program will run if the code is correct.

Type 1 Conditionals

Type 1 conditionals talk about a real or very probable situation in the present or future. Here is the formula:

If-clause Main clause
If + present simple future simple (will)
present simple
imperative (a command)
modal verb (can/must/may) + bare infinitive
  • If it snows in Lviv this winter, we will build a snowman.
  • If you mix yellow and blue, you get green.
  • If you don't understand the math problem, ask the teacher!
  • If you go to a cafe, you can order a hot chocolate.

We can use the word unless instead of if...not. The verb after "unless" is always positive. Think of it like saying "except if".

  • If you don't fix the bug, the game won't work.
  • Unless you fix the bug, the game won't work.

Conditionals – Type 2

Type 2 Conditionals

Type 2 conditionals are for dreaming! We use them to talk about imaginary or improbable situations that are unlikely to happen in the present or future. Here is the formula:

If-clause Main clause
If + past simple would/could/might + bare infinitive
  • If I had a million dollars, I would travel the world.
  • If I knew how to code a 3D game, I would make a masterpiece.

In Type 2 Conditionals, we can use were instead of was for all persons. It sounds more formal and correct.

  • If Peter was here, he would fix the computer.
  • If Peter were here, he would fix the computer. (Preferred!)

We use the phrase If I were you... when we want to give someone advice.

  • If I were you, I’d ask the teacher to explain the math formula again.
